The `git init` command creates a new Git repository. It can be used to convert an existing, unversioned project to a Git repository or initialize a new, empty repository.

## Why Use It

This is the first command you will run in a new project. Running `git init` creates a `.git` subdirectory in your current working directory, which contains all of the necessary metadata for the repo.

## Syntax

```bash
git init [directory]
```

## Examples

**Initialize in the current directory**
```bash
git init
```

**Initialize in a new directory**
```bash
git init my-project
```

## Common Use Cases
- Starting a brand new project.
- Taking an existing folder of code and putting it under version control.
