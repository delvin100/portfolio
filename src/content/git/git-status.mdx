# git status

## Description

The `git status` command displays the state of the working directory and the staging area. It lets you see which changes have been staged, which haven't, and which files aren't being tracked by Git. 

It does **not** show you any information regarding the committed project history. For that, you need to use `git log`.

## Why Use It

You should use `git status` constantly. It is your primary tool for understanding what is going on with your files before you run `git add` or `git commit`. If you are ever confused about what state your repository is in, `git status` is the first command to run.

## Syntax

```bash
git status [options]
```

## Parameters

- `-s`, `--short`
  Give the output in the short-format.
  
- `-b`, `--branch`
  Show the branch and tracking info even in short-format.
  
- `--ignored`
  Show ignored files as well.
  
- `-u[<mode>]`, `--untracked-files[=<mode>]`
  Show untracked files. Mode can be `no` (show no untracked files), `normal` (shows untracked files and directories), or `all` (also shows individual files in untracked directories).

## Examples

**Basic Status Check**
```bash
git status
```

**Short Format**
If you have a lot of files and want a more compact view:
```bash
git status -s
```

## Example Output

```text
On branch main
Your branch is up to date with 'origin/main'.

Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git restore <file>..." to discard changes in working directory)
        modified:   src/app/globals.css
        modified:   src/content/git/git-status.mdx

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        src/components/docs/

no changes added to commit (use "git add" and/or "git commit -a")
```

## Common Use Cases

- Checking which files you have modified before adding them.
- Verifying you haven't forgotten to stage any new files.
- Seeing if your local branch is ahead or behind the remote branch.

## Common Mistakes

- **Forgetting to read the output:** Git's status messages are extremely helpful and usually tell you exactly what command you need to run next (like `git add` or `git restore`). Beginners often ignore this text!

## Tips

> Run `git status` before and after every major Git command so you always know exactly what state your repository is in.
