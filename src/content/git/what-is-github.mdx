If Git is the engine, GitHub is the parking garage. 

While **Git** is the actual software running on your local computer that tracks your files, **GitHub** is a website (a cloud hosting service) that allows you to store your Git repositories online.

## Why use GitHub?

1. **Backup:** If your computer crashes and you lose all your files, your code is safely backed up in the cloud.
2. **Collaboration:** Your teammates cannot easily download code directly from your laptop. GitHub acts as a central hub where everyone can push and pull code.
3. **Open Source:** It is the largest community of developers in the world. You can share your code publicly and contribute to other people's projects.

## Git vs GitHub

It is very common for beginners to confuse the two. Remember:

* **Git** is the version control system. It works completely offline on your computer. You don't need the internet to use Git.
* **GitHub** is a company (owned by Microsoft) that hosts Git repositories on the internet. 
* There are other companies that do the exact same thing as GitHub, such as **GitLab** and **Bitbucket**.
